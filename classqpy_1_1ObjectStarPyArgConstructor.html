bool qpy::ObjectStarPyArgConstructor::IsQObjectPtr ( ) const [inline, virtual]

Return true if type is a pointer to a QObject-derived object.

This is required to have the QPy run-time add the passed QObject into the Python context. The other option is to have PyArgConstructors::Create receive a reference to a PyContext which introduces a two-way dependency between PyArgConstructor and PyContext.

Reimplemented from qpy::PyArgConstructor.
